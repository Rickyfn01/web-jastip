# JastipVIP - User Journey & System Architecture

## 📋 Ringkasan Sistem

**JastipVIP** adalah platform personal shopper premium yang menghubungkan calon pembeli dengan produk brand ternama menggunakan harga "diskon karyawan" yang eksklusif. Sistem ini dibangun dengan Next.js 14, PostgreSQL, dan terintegrasi dengan **Xendit** untuk payment gateway yang aman.

---

## 🛍️ ALUR PEMBELI (Customer Journey)

### **Fase 1: Discovery & Request**

**Lokasi:** Landing Page (`/`)

| Elemen | Detail |
|--------|--------|
| **Entry Point** | User membuka `jastipvip.vercel.app` |
| **CTA** | Tombol "Titip Barang Sekarang" atau form request langsung |
| **Input Form** | - Upload foto barang (drag & drop)<br/> - Nama barang<br/> - Brand (Nike, Zara, Sephora, dll)<br/> - Ukuran/Size<br/> - Warna<br/> - **Pilihan DP: 50%, 75%, atau 100%** |
| **Sistem Action** | Order dibuat dengan status `PENDING` di database |
| **Customer Gets** | 1. Order ID unik<br/> 2. Notifikasi WhatsApp dengan tracking link<br/> 3. Akses ke halaman tracking pribadi |

**Database State:**
```
Order {
  status: PENDING
  customerName: [dari form]
  whatsappNumber: [dari form]
  productImage: [URL dari Supabase]
  dpPercentage: [50|75|100]
  createdAt: now()
}
```

### **Fase 2: Tracking & Monitoring**

**Lokasi:** `/track/[order-id]`

**Status: PENDING → CHECKING**

| Elemen | Detail |
|--------|--------|
| **Yang dilihat customer** | - Order ID<br/> - Status: "Menunggu Konfirmasi"<br/> - Foto barang yang diminta<br/> - Admin sedang mencari ke store |
| **Durasi typical** | 1-3 jam (depends on stock) |

**Status: CHECKING**

| Elemen | Detail |
|--------|--------|
| **Yang dilihat customer** | - Status: "Sedang Dicari"<br/> - Countdown timer<br/> - Message: "Admin sedang cek ke store..." |
| **Admin Action** | Admin checking ke store/supplier, foto barang |

---

### **Fase 3: Payment Ready**

**Lokasi:** `/track/[order-id]` atau `/member/dashboard`

**Status: READY_TO_PAY (Barang Ditemukan)**

| Elemen | Detail |
|--------|--------|
| **Xendit Integration** | - Xendit Invoice **otomatis di-generate**<br/> - Invoice Link dikirim ke tracking page<br/> - Expiry: **24 jam** dari generation time |
| **Yang dilihat customer** | - Status: "Barang Ditemukan - Siap Bayar"<br/> - **Final Price ditampilkan** (StorePrice + Jastip Fee Rp25k)<br/> - **DP Amount** yang harus dibayar sekarang<br/> - Countdown: "Invoice berlaku untuk XX jam XX menit"<br/> - Tombol: "Bayar DP Sekarang" |
| **Harga Calculation** | **Final Price** = (Store Price - Diskon Admin) + Rp25,000<br/> **DP Amount** = Final Price × (dpPercentage / 100) |
| **Example** | Store: Rp800k<br/> Discount: 20% (Rp160k)<br/> Fee: Rp25k<br/> **Final: Rp665k**<br/> DP 50%: Rp332,500 |

**Database State:**
```
Order {
  status: READY_TO_PAY
  storePrice: [input by admin]
  discountPercent: [input by admin]
  finalPrice: (storePrice - discount) + 25000
  dpAmount: finalPrice * (dpPercentage / 100)
  xenditInvoiceId: [generated by Xendit]
  xenditInvoiceUrl: [payment link]
  xenditInvoiceExpiryAt: now + 24 hours
  receiptImage: [optional, waiting for admin upload]
}
```

---

### **Fase 4: Payment Processing**

**Lokasi:** Xendit Payment Gateway / Member Dashboard

**Action: Customer clicks "Bayar DP Sekarang"**

| Elemen | Detail |
|--------|--------|
| **Payment Method** | - Virtual Account (BCA, Mandiri, BNI)<br/> - e-Wallet (OVO, Dana, LinkAja)<br/> - QRIS |
| **Invoice Details** | - Amount: DP Amount<br/> - External ID: `order-dp-{orderId}`<br/> - Item Description: "DP Pembelian: [Brand] [Barang]"<br/> - Callback URL: `/api/webhooks/xendit` |
| **Customer Experience** | - Dipandu step-by-step di Xendit gateway<br/> - Real-time payment confirmation |

**Flow:**
```
Customer Pay DP 
    → Xendit verifies payment
    → Hits webhook /api/webhooks/xendit
    → System updates Order status to PAID_DP
    → dpPaidAt timestamp recorded
    → dpAmount marked as paid
```

| **Customer Sees** | - Status badge: "✅ DP Sudah Dibayar"<br/> - Amount yang sudah dibayar<br/> - Sisa pembayaran (Final - DP)<br/> - Button: "Saya Sudah Bayar, Refresh Status"<br/> - Optional: Chat admin button |
| **Refresh Mechanism** | Customer bisa click "Refresh Status" button untuk sync terbaru (fetch dari `/api/customers/[id]`) |

**Database State:**
```

  status: PAID_DP
  dpPaidAt: [timestamp dari webhook]
  xenditInvoiceStatus: PAID
  receiptImage: [waiting for admin evidence]
}
```

### **Phase 3: Price Calculation (Status: READY_TO_PAY)**

| Element | Detail |
|---------|--------|
| **Action** | Admin fills price form in dashboard |
| **Input Fields** | 1. **Store Price** (harga asli dari toko)<br/> 2. **Discount %** (jastip profit margin, e.g., 15%, 20%)<br/> 3. **Jastip Fee** (fixed Rp25,000) |
| **Auto-Calculation** | **Final Price** = (Store Price - Discount) + Jastip Fee<br/> **DP Amount 50%** = Final Price × 0.5<br/> **DP Amount 75%** = Final Price × 0.75<br/> **DP Amount 100%** = Final Price × 1.0 |
| **Display to Admin** | Real-time breakdown showing:<br/> - Store Price: Rp [amount]<br/> - Discount (20%): - Rp [amount]<br/> - Jastip Fee: + Rp 25,000<br/> - **Final Price: Rp [amount]** (bold)<br/> - **DP 50%: Rp [amount]** (atau 75/100 sesuai pilihan customer) |
| **Action** | Admin click "Confirm Price" |

**Database State After Confirmation:**
```
Order {
  status: READY_TO_PAY
  storePrice: [user input]
  discountPercent: [user input]
  finalPrice: calculated
  dpAmount: calculated
  xenditInvoiceId: [auto-generated by Xendit SDK]
  xenditInvoiceUrl: [from Xendit response]
  xenditInvoiceExpiryAt: now + 24 hours
}
```

**Xendit Trigger:**
```
When status changes to READY_TO_PAY:
1. Backend calls Xendit SDK: createDpInvoice()
2. Xendit creates invoice with:
   - Amount: dpAmount
   - ExternalId: `order-dp-{orderId}`
   - Description: "DP Pembayaran: [Customer] - [Brand]"
   - DueDate: 24 hours from now
   - Customer email/name
3. Receives response with:
   - invoiceId
   - invoiceUrl (payment link)
   - expiryDate
4. Saves to Order: xenditInvoiceId, xenditInvoiceUrl, xenditInvoiceExpiryAt
5. Customer gets payment link via tracking page
```

## 💾 DATABASE SCHEMA

### Order Model

```prisma
model Order {
  id                        String      @id @default(uuid())
  
  // Customer Info
  customerId                String?
  customer                  Customer?   @relation(fields: [customerId], references: [id])
  customerName              String
  whatsappNumber            String
  
  // Product Details
  productImage              String?     // Supabase URL
  productNote               String?
  brand                     String
  size                      String?
  color                     String?
  
  // Status & Timeline
  status                    Status      @default(PENDING)
  createdAt                 DateTime    @default(now())
  updatedAt                 DateTime    @updatedAt
  purchasedAt               DateTime?
  
  // Pricing
  storePrice                Float?      // Harga toko asli
  discountPercent           Float?      // Diskon jastip
  finalPrice                Float?      // (StorePrice - Discount) + 25000
  dpPercentage              Int         // 50, 75, atau 100
  dpAmount                  Float?      // finalPrice * (dpPercentage / 100)
  
  // Payment - DP (Down Payment)
  dpPaidAt                  DateTime?
  
  // Xendit Integration
  xenditInvoiceId           String?     @unique
  xenditInvoiceUrl          String?
  xenditInvoiceStatus       String?     // PENDING, PAID, EXPIRED
  xenditInvoiceExpiryAt     DateTime?   // Invoice deadline
  
  // Evidence
  receiptImage              String?     // Struk belanja (Supabase URL)
  
  @@index([customerId])
}

enum Status {
  PENDING
  CHECKING
  READY_TO_PAY
  PAID_DP
  PURCHASED
  SHIPPED
  COMPLETED
}
```

## 🔐 Xendit Integration Details

### Invoice Creation Trigger

```
When: Order status changes to READY_TO_PAY
Where: /src/lib/xendit.ts → createDpInvoice()
What: Creates invoice via Xendit SDK
```

### Invoice Request
```javascript
{
  amount: dpAmount,
  invoiceDueDate: now + 24 hours,
  customerName: customerName,
  customerEmail: customerEmail,
  externalId: `order-dp-${orderId}`,
  description: `DP Pembelian: ${brand} ${productName}`,
  items: [{ name: productName, amount: dpAmount }]
}
```

### Webhook Handler
```
POST /api/webhooks/xendit
- Validates X-Callback-Token header
- Extracts orderId from externalId
- Updates order status based on invoice status:
  • PAID → status = PAID_DP, dpPaidAt = now
  • EXPIRED → status = READY_TO_PAY (invoice expires, needs renewal)
- Returns 200 OK
```

### Invoice Expiry Handling
```
- Invoice valid: 24 hours from creation
- If not paid: Status remains READY_TO_PAY
- Customer sees: "Invoice kadaluarsa, silakan hubungi admin"
- Admin needs to: Create new invoice (auto via READY_TO_PAY trigger)
```
